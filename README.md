# woowa-mission-lint

`woowa-mission-lint`는 우아한 테크코스 프리코스 미션 진행을 돕는 ESlint/Prettier 통합 검증 라이브러리 입니다.

## 왜 필요할까요?

주차를 거듭하면서 요구사항이 추가되고, 파일도 많아지면서 놓친 요구사항이 있지는 않으셨나요?

ESLint와 Prettier를 각각 설치하고, 여러개의 설정파일을 만들고 수정하느라 고생하시진 않으셨나요?

영어로 된 설정 파일을 수정해야 하는데, 어디서부터 건드려야 할 지 어려움을 겪지 않으셨나요?

**프리코스를 진행하며 요구사항을 검증할 때 겪으셨던 어려움을 해결해 제가 해결해 드릴게요** 😊

## 사용하기

1. 모듈 다운로드

```sh
   npm i woowa-mission-lint
```

2. 설정파일 생성

```sh
    npx woowa-lint --init
```

woowalint.json 파일이 생성됩니다.

3. 추가 설정 없이 사용하기

   - 레벨 1

   ```sh
   npx woowa-lint --level 1
   ```

   - 레벨 2

   ```sh
   npx woowa-lint --level 2
   ```

   - 레벨 3

   ```sh
   npx woowa-lint --level 3
   ```

   각 Level의 규칙은 [여기서](#레벨별-추가-규칙) 확인해주세요

4. 사용자 정의 설정 적용하기

```js
// woowalint.json

{
  "활성_레벨": "level_1",
  "레벨별_규칙": {
    "level_1": {
      /* ... level 1 규칙 ... */
    },
    "level_2": {
      /* ... level 2 규칙 ... */
    },
    "level_3": {
      /* ... level 3 규칙 ... */
    }
  }
}
```

처음 생성된 `woowalint.json`파일은 사전에 정의된 레벨별 규칙이 **한글**로 작성되어 있습니다.

기존 level 1~3의 규칙을 수정할 수 있습니다.
필요하다면 `level_4 : {사용자 정의 규칙}`과 같은 커스텀 레벨을 생성하고
`npx woowa-lint --level 4`를 통해 커스텀 레벨 규칙을 적용할 수 있습니다.

사용 가능한 규칙 목록은 [여기서](#사용-가능한-규칙-목록) 확인해주세요

## Lint 수준과 사용자 정의 설정

### `woowa-mission-lint`의 기본 코드 스타일

모든 코드의 스타일을 일관되게 유지하기 위해 다음과 같은 Prettier 규칙을 강제합니다. (별도 설정 필요 없음)

1.  문자열을 사용할 때는 **작은 따옴표('')** 사용
2.  코드의 각 라인 끝에 **세미콜론(;)** 삽입
3.  들여쓰기는 **space 2칸** 사용
4.  한 줄의 최대 글자수는 **80자**
5.  화살표 함수의 인자가 1개일때도 괄호 사용 강제
6.  객체나 배열의 마지막 항목 이후에도 **콤마(,)** 사용
7.  `if`, `for`, `while` 문을 사용할 때 공백 추가

### `woowa-mission-lint`의 기본 린트 규칙

1. if문을 사용할 때는 중괄호를 생략할 수 없습니다.
2. 변수명/함수명은 **camelCase**를 준수해야 합니다.
3. `constants` 폴더 내의 상수는 **SNAKE_CASE**를 준수해야 합니다.
4. console.log를 사용할 수 없습니다.
5. 메서드 사이에는 1줄의 공백이 들어갑니다.

### 레벨별 추가 규칙

#### Level 1

- 기본 린트 규칙만 적용됩니다.

#### Level 2

- Level 1의 모든 규칙이 적용됩니다.
- **indent depth(들여쓰기)가 2를 넘지 않는지 확인합니다.**
- **삼항연산자를 사용하지 않았는지 확인합니다.**

#### Level 3

- Level 2의 모든 규칙이 적용됩니다.
- **한 메서드가 15줄을 넘지 않는지 확인합니다.**

| ------------------------------ | ------- | ------------------------------------ |

| "메서드\_사이\_공백"           | Boolean | "메서드\_사이\_공백": true           |

| "if문\_중괄호\_필수"           | Boolean | "if문\_중괄호\_필수": true           |
